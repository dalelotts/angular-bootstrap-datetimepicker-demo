This example combines an input, dl-date-time-input directive, a bootstrap dropdown, and the dl-date-time-picker component.

The dl-date-time-picker is completely unaware that it is contained in a dropdown, so you will need to write some code to close the dropdown when a date is selected by the user. This is an intentional design decision because it allows the developer using this component to use any dropdown they like.

Specifically, when using the bootstrap dropdown, the code will have to track the open state of the dropdown by listening for show.bs.dropdown and hide.bs.dropdown events emitted by bootstrap's dropdown.

Nota bene The interactions between keyboard entry of the date, the date-picker selection of a date, and validation of the both input sources is not included in this example. The simplest solution is to avoid keyboard input; if that will work for your use case.
